## ðŸ”’ SAFETY & SECURITY (CRITICAL)

### The Scary Incident
**What happened:**
- Developer used Turbo Mode
- Agent started running terminal commands
- **Accidentally deleted entire hard drive**
- No warning
- Files just gone

**Why it happened:**
- Agents have full access to terminal
- Can run any command (including `rm -rf /`)
- Turbo mode skips approval steps

### Security Vulnerabilities Found
**Prompt injection attacks:**
- Bad actors hide malicious instructions in files
- When AI reads files, executes instructions
- Could steal API keys
- Could delete databases
- Could expose sensitive data

---

### Safety Rules (Julian's Recommendations)

**Rule 1:** Always back up your work
- Use Git for version control
- Commit frequently
- Push to remote (GitHub, GitLab)

**Rule 2:** Run in sandbox or VM
- Never run on production servers
- Use dedicated virtual machine
- Isolate from critical systems

**Rule 3:** Disable automatic destructive commands
- No automatic `rm -rf`
- No global `sudo` access
- Require manual approval for dangerous commands

**Rule 4:** Review evidence logs
- Check what agent did
- Read through artifacts
- Don't blindly trust

**Rule 5:** Use dummy credentials for testing
- Don't give real API keys to agents
- Don't connect to production databases
- Use test environments

**Rule 6:** Avoid Turbo Mode on important work
- Turbo mode = faster but dangerous
- Skips approval steps
- Only use for throwaway projects

**Julian's warning:**
> "Anti-gravity agents have a lot of power. They can edit code. They can run terminal commands. They can control your browser. This creates security risks."
